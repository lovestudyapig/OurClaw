# 定时任务管理技能 (task_manager)

## 使用场景
当用户要求创建定时任务、定时执行某项功能时，使用此技能。例如：
- "帮我创建一个每天定时检查服务器状态的定时任务"
- "每天早上8点帮我发送天气提醒"
- "我要一个每10分钟检测一次温度的任务"

## 前置知识

### 任务文件格式规范
每个定时任务是一个独立的 `.py` 文件，保存在 `tasks/` 文件夹下，必须包含以下内容：

```python
"""任务描述文档字符串。"""

import asyncio
from datetime import datetime

# 任务元数据（必需）
name = "任务名称"            # 任务显示名称
interval = 60                # 执行间隔（秒）
description = "任务描述"     # 任务功能描述

async def run():
    """任务执行函数（必需）。"""
    # 任务逻辑
    now = datetime.now().strftime("%H:%M:%S")
    print(f"[任务:名称] {now} - 执行内容")
```

### 任务调度机制
- `background_loop.py` 中的 `BackgroundLoop` 负责调度所有任务
- 启动时通过 `scan_tasks()` 扫描 `tasks/` 文件夹动态加载所有 `.py` 任务文件
- 每个任务独立运行，互不影响
- 任务可以随时通过 `register()` 方法动态注册

### 可用工具
创建任务时可使用以下工具：
- `terminal` — 执行 shell 命令，调用系统功能
- `python_repl` — 执行 Python 代码进行数据处理
- `fetch_url` — 抓取网页内容、访问API
- `tavily_search` — 联网搜索获取实时信息
- `read_file` — 读取文件内容
- `write_memory` — 记录信息到记忆系统

## 执行步骤

### Step 1：分析用户需求
仔细理解用户想要创建的定时任务：
- 任务功能：具体要做什么
- 执行频率：间隔时间（秒/分钟/小时）
- 任务名称：便于识别的名称
- 输入输出：是否需要访问外部数据、是否需要保存结果

### Step 2：提取关键信息并确认
向用户确认以下信息：
- 任务名称（英文/中文均可）
- 执行间隔（明确指定秒数）
- 任务的具体逻辑

### Step 3：编写任务代码
按照上述任务文件格式规范，编写 Python 代码：
1. 包含完整的文档字符串
2. 定义 `name`、`interval`、`description` 元数据
3. 实现 `async def run()` 函数
4. 确保代码健壮，包含适当的异常处理

### Step 4：保存任务文件
将任务文件保存到 `tasks/` 目录下：
- 文件名：`[任务名称].py`（使用英文小写+下划线）
- 路径：`tasks/[任务名称].py`

### Step 5：测试任务
1. 手动运行一次任务函数，验证逻辑正确
2. 使用 `python_repl` 执行 `import importlib; import sys; ...` 动态导入并调用 `run()` 验证
3. 确保无语法错误和运行时错误

### Step 6：输出结果
告知用户：
- 任务文件保存路径
- 任务名称和执行间隔
- 任务已注册，将在下次 `BackgroundLoop` 启动时自动加载
- 可通过 `loop.register()` 动态注册立即启动

## 注意事项
- 任务文件必须放在 `tasks/` 目录下才能被自动扫描加载
- 任务函数必须是 `async def` 异步函数
- 异常必须在任务内部捕获处理，避免崩溃影响其他任务
- 间隔时间不宜过短（建议 >= 5秒），避免资源浪费
- 文件名不要以 `_` 开头（会被扫描器跳过）
